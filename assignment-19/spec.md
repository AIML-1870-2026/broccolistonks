# Science Experiment Generator — spec.md

## Overview

A single-file (`index.html`) web app that lets users generate grade-appropriate science experiments using the OpenAI API. The user selects a grade level, enters available supplies, and receives a formatted experiment generated by an LLM. Designed for GitHub Pages deployment — no backend server.

---

## Design Constraints

- **OpenAI models only** — no Anthropic or Google provider switching. The OpenAI API allows direct browser-to-API calls (no CORS issues).
- **Unstructured responses** — the model returns free-form text, not JSON.
- **Markdown rendering** — render the model's markdown response as properly formatted HTML (bold, lists, headings, etc.). Use a lightweight library like `marked` via CDN.
- **API key in memory only** — user pastes their OpenAI API key into an input field at the top of the page. It is stored in a JS variable, never persisted to localStorage or anywhere else.
- **Single-file deployment** — everything (HTML, CSS, JS) lives in one `index.html` file. External CDN scripts (e.g. `marked`) are fine.

---

## Phase 1 — Layout & Static UI

Build the full page structure with no API wiring yet. Everything should look complete and be interactive (dropdowns open, buttons exist) but submitting does nothing.

### Requirements

1. **API Key input** — a text input at the top for the user to paste their OpenAI key. Include a show/hide toggle.
2. **Model selector** — a dropdown to pick an OpenAI model. Include at least: `gpt-4o`, `gpt-4o-mini`, `gpt-3.5-turbo`.
3. **Grade Level dropdown** — options: `K-2`, `3-5`, `6-8`, `9-12`.
4. **Available Supplies** — a text input (or textarea) where the user types a comma-separated list of supplies they have on hand.
5. **Generate button** — prominent call-to-action.
6. **Result area** — an empty container below the form where the experiment will be rendered.
7. **Visual design** — clean, friendly, science-themed. Use a distinctive font pairing from Google Fonts (not Inter/Roboto). Pick a cohesive color palette that feels educational but not childish. Add subtle background texture or gradient for atmosphere.
8. **Responsive** — works on desktop and mobile.

---

## Phase 2 — OpenAI API Integration

Wire up the Generate button to call the OpenAI Chat Completions API and display the result.

### Requirements

1. On click of Generate:
   - Validate that the API key, grade level, and at least one supply are provided. Show inline error messages if not.
   - Show a loading spinner or skeleton in the result area.
   - Call `https://api.openai.com/v1/chat/completions` with:
     - `model`: whatever is selected in the dropdown.
     - `messages`: a system prompt + user prompt.
   - **System prompt** (suggested): `"You are a creative science education expert. Generate a single, detailed, hands-on science experiment appropriate for the given grade level using only the supplies provided. Include: a fun title, objective, list of materials used, step-by-step instructions, the scientific concept being explored, and expected results. Use markdown formatting."`
   - **User prompt**: `"Grade level: {grade}. Available supplies: {supplies}."`
2. On success, parse the response and render the markdown content as HTML using `marked` (loaded from CDN).
3. On error (bad key, network issue, rate limit), display a user-friendly error message in the result area.
4. Disable the Generate button while a request is in flight to prevent duplicate calls.

## Phase 3 — Stretch Features

Add at least two of the following enhancements. Pick whichever feel most impactful.

### Options (implement 2+)

- **Saved experiments** — add a "Save" button on each generated experiment. Store saved experiments in a list on the page (in-memory array, lost on refresh is fine). Let the user click a saved experiment to view it again.
- **Predefined supply chips** — below the supplies input, show clickable chips/tags for common household items (vinegar, baking soda, food coloring, cups, balloons, etc.). Clicking a chip adds it to the input.
- **Difficulty rating** — after the experiment is generated, display a difficulty badge (Easy / Medium / Hard) based on the grade level or ask the model to include one.
- **Print / Export** — a button to print the experiment or copy it as markdown to clipboard.
- **Supply substitution** — if the user is missing a supply, a secondary button asks the model to suggest alternatives.

## Tech Notes for Claude Code

- Load `marked` from CDN: `<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>`
- The fetch call to OpenAI looks like:
  ```js
  const res = await fetch("https://api.openai.com/v1/chat/completions", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "Authorization": `Bearer ${apiKey}`
    },
    body: JSON.stringify({
      model: selectedModel,
      messages: [
        { role: "system", content: systemPrompt },
        { role: "user", content: userPrompt }
      ]
    })
  });
  const data = await res.json();
  const markdown = data.choices[0].message.content;
  ```
- The result text is in `data.choices[0].message.content`.
- Render with `marked.parse(markdown)` and insert as `innerHTML`.
- All CSS and JS go inside the single `index.html` file (use `<style>` and `<script>` tags).
- Do NOT use localStorage or sessionStorage for the API key. Keep it in a JS variable only.
